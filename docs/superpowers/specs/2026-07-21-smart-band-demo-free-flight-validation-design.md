# Smart Band Demo：随心飞验证与可迁移底座设计

## 决策

Smart Band Demo 是 WearableKit 的 Android 验证宿主，不是小睡眠正式产品的前身。它保留手环调试所需的绑定、日志、模拟和设备差异；验证成熟后，只有通用能力与契约被移植到小睡眠 App。

原有 Web 随心飞 Demo 继续作为动画、音频和交互效果的参考，不再承担真实硬件链路验证。

此前内部使用的 PsyWatch 名称不作为后续产品/模块名称。本设计统一称为 WearableKit；如仍调用旧算法库，将它视为 WearableKit 的内部实现细节。

## 页面架构

Smart Band Demo 顶部保留三个平级 Tab：

| 页面 | 定位 | 主要职责 | 不承担 |
|---|---|---|---|
| 连接 | 硬件与数据源调试 | 绑定、授权、设备能力、连接状态、实时数据通道 | 随心飞会话与健康结论 |
| 睡眠 | 离线睡眠数据验证 | 厂商睡眠读取、本地/厂商数据验证、后续标准化融合输入 | 实时干预 |
| 随心飞验证 | 实时闭环验证 | 一次完整的实时会话、状态反馈、干预、异常降级、会话摘要 | 正式产品信息架构、全天压力结论 |

第三页的名称固定为“随心飞验证”，明确它服务于研发验收，而非面向普通用户的最终页面。

## 共享边界

### 留在 Demo 的验证层

- Tab、设备厂商选择、扫描/连接细节、原始包与调试日志。
- 手动模拟状态、强制断连、fixture 回放、开发开关。
- 用于定位问题的原始样本可视化和临时诊断文案。

### 进入 WearableKit 的可迁移层

- 设备数据的来源、覆盖率、样本数、连接状态和质量判定。
- 实时会话生命周期：准备、采样、运行、降级、结束、中断。
- 状态节点输入/输出契约，默认会话的时间节点为 30、90、150、210、270 秒。
- 随心飞的内容配置读取、资源 ID 映射、状态到反馈组映射、干预次数/冷却规则。
- FreeFlightSessionSummary：会话结果、来源、质量、版本、状态趋势、干预完成情况和下一步入口。
- 后续正式 HRV 能力所需的原始 RRI/PPI 来源标记与有效性判断。

### 移植到小睡眠 App 的产品层

- 正式随心飞入口、动画表现、音频、结果页与睡眠监测跳转。
- 晨起睡眠报告、今日报告及其推荐文案。
- 用户授权、数据保留、账号同步和正式埋点。

## 随心飞验证页设计

### 启动前

页面顶部显示：

- 当前绑定设备、实时通道状态和数据来源。
- 场景选择：日间或睡前。
- 验证模式：真实设备、fixture 回放、无设备降级。
- 开始按钮；未满足真实设备前提时，明确说明原因，并允许切换至可用验证模式。

### 运行中

会话默认 5 分钟，前 30 秒为采样期，之后按五个状态节点刷新。页面采用沉浸式表现区，但保留小型验证信息层：

- 飞行轨迹/动画、音频组和当前状态反馈。
- 心率仅作为实时信号展示；无数据或断连显示 --。
- 当前节点、倒计时、来源和质量标签。
- 本次是否已触发呼吸干预；默认 5 分钟最多一次。
- 开发态可展开诊断抽屉，查看样本数、覆盖率、状态置信度、内容版本和断连原因。

状态反馈组为：专注平静、兴奋高兴、疲倦沮丧、焦虑紧张。状态缺失时沿用上一次有效状态；无设备或断连时使用默认资源和安全降级体验。

### 结束后

展示会话摘要而非医疗诊断：

- 会话是否完成，持续时间，数据来源和质量。
- 最后有效状态及趋势、干预是否完成。
- 下一步：再次验证、开始睡眠监测、返回。
- 调试态可复制结构化摘要。

## 数据与状态流

    WearableKit 原始实时数据
      -> 数据源/质量判定
      -> WearableKit 内部状态计算
      -> 随心飞会话状态机
      -> 动画、音频、呼吸引导
      -> FreeFlightSessionSummary
      -> Demo 日志 / 未来小睡眠随心飞与今日报告

正式 HRV 字段只能来自已经验证为原生峰值间期的 PPI/RRI。5 秒或 7 秒心率、估算 RRI、厂商压力分可以用于探索和调试展示，但不得填入正式 RMSSD、正式 HRV 或正式压力结论。

摘要至少包含：会话 ID、开始/结束时间、场景、完成状态、数据来源、质量、内容/算法/客户端版本、状态节点、心率摘要、可空的 HRV 摘要、干预记录和下一步动作。

## 异常与降级

| 情况 | 体验 | 数据规则 |
|---|---|---|
| 未绑定或无实时数据 | 10 秒提示后进入无设备体验 | source=no_wearable，quality=insufficient |
| 中途断连 | 心率显示 --，保持默认播放，可结束 | 不产生正式 HRV/压力结论 |
| 样本不足 | 状态不频繁跳变，展示“未形成有效结论” | quality=insufficient |
| 厂商探索数据 | 可注明来源并用于调试展示 | quality=exploratory，正式 HRV 字段为空 |
| 内容配置失效 | 使用内置已验证配置 | 保留失败日志和内容版本 |

## 验收顺序

1. WearableKit：绑定后实时数据可读、断连可感知、来源和质量正确。
2. 验证页无设备闭环：五分钟计时、默认音频/轨迹、结束摘要。
3. fixture 回放：五个状态节点、状态保持、一次呼吸干预、音频转场和资源冷却。
4. 真实手环会话：有效、样本不足、断连三类结果正确。
5. 迁移准备：摘要和配置不依赖 Demo UI，可由小睡眠 App 消费。

## 不在本轮

- 把 Demo 的页面、调试日志或厂商选择界面直接搬进小睡眠 App。
- 全天连续压力分、医疗诊断或使用低频心率伪造 HRV。
- 重做正式睡眠报告或今日报告；本轮仅预留会话摘要的可迁移契约。
